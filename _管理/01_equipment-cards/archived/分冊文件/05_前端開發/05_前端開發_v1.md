# 第 05 冊：前端開發

## 1. 專案架構

技術選型：Next.js 14（App Router）+ Tailwind CSS + shadcn/ui

```
src/
├── app/
│   ├── layout.tsx              ← Root layout（Inter 字型、全域 CSS）
│   ├── page.tsx                ← 首頁（Server Component）
│   ├── globals.css             ← Tailwind v3 + shadcn CSS 變數
│   └── api/upload/route.ts     ← 照片上傳 API Route
├── components/
│   ├── PhotoWall.tsx           ← 搜尋 + 篩選 + 卡片網格（Client Component）
│   ├── EquipmentCardItem.tsx   ← 單張卡片縮圖
│   ├── CardDetailDialog.tsx    ← 細節 Lightbox（照片輪播 + 資訊欄）
│   └── ui/                     ← shadcn/ui 元件（Badge、Dialog、Input 等）
├── hooks/
│   └── usePhotoUpload.ts       ← 照片上傳 hook（供管理後台使用）
├── lib/
│   ├── supabase.ts             ← Supabase client（anon key，前端用）
│   └── utils.ts                ← cn() Tailwind class merge helper
└── types/
    └── equipment.ts            ← EquipmentCard TypeScript 型別定義
```

## 2. 資料流架構

```
Supabase（資料庫）
    ↓ Server Component 在 request 時撈取全部資料
page.tsx（Server Component）
    ↓ 透過 props 傳入
PhotoWall.tsx（Client Component）
    ↓ Fuse.js 過濾後渲染
EquipmentCardItem × N（卡片網格）
    ↓ 點擊
CardDetailDialog（Lightbox）
```

首頁採用 Server Component + Client Component 混合設計：
- `page.tsx` 在伺服器端直接呼叫 Supabase（使用 Service Role Key），一次撈取全部 786 筆
- 資料傳給 `PhotoWall` 後，後續搜尋／篩選全在客戶端由 Fuse.js 處理，不需要再打 API

## 3. TypeScript 型別（equipment.ts）

```typescript
export interface DetailPhoto {
  public_id: string   // Cloudinary public_id，刪除時使用
  url: string         // Cloudinary secure_url
}

export interface EquipmentCard {
  equipment_id: string
  name: string
  category: string | null
  vendor: string | null
  status: 'active' | 'discontinued'
  tags: string[]
  notes: string | null
  main_photo: string | null
  main_photo_public_id: string | null
  detail_photos: DetailPhoto[]
  created_at: string
  updated_at: string
}
```

## 4. Supabase Client（supabase.ts）

前端使用 anon key（有 RLS 保護），只能讀取資料：

```typescript
export const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
)
```

> `page.tsx` 的 Server Component 改用 `SUPABASE_SERVICE_ROLE_KEY`（不暴露給前端，繞過 RLS）

## 5. 首頁（page.tsx）

伺服器端撈取資料，`useSearchParams()` 在 Client Component 中使用時需要用 `<Suspense>` 包裝，否則 Next.js build 會報錯。

## 6. 卡片元件（EquipmentCardItem.tsx）

每張卡片顯示：
- 縮圖：`main_photo`，用 Next.js `<Image>` 元件（自動 WebP 轉換、lazy load）
- 停產遮罩：`status === 'discontinued'` 時疊加黑色半透明遮罩 + 紅色「停產」badge
- 細節照片數：右下角顯示 `+N`（detail_photos 數量）
- 品號、品名、廠商、標籤（最多顯示 2 個 tag，超過顯示 +N）

Cloudinary 圖片網域需在 `next.config.mjs` 加入白名單：

```javascript
images: {
  remotePatterns: [{ protocol: 'https', hostname: 'res.cloudinary.com', pathname: '/**' }]
}
```

## 7. 細節 Dialog（CardDetailDialog.tsx）

點擊卡片後開啟，左右分割版面：

左側（照片區）：
- 照片陣列 = `[main_photo, ...detail_photos]`
- 左右箭頭切換 + 底部縮圖列 + 指示點
- 照片標籤：「主圖（1/N）」「細節 1（2/N）」…

右側（資訊區）：
- 品號、品名、狀態 Badge
- 分類、廠商
- 標籤（全部顯示）
- 備註（多行保留換行）
- 照片數量統計

## 8. 照片上傳 API Route（api/upload/route.ts）

採用 Cloudinary Signed Upload，避免 API Secret 暴露給前端，且檔案不經過 Vercel 伺服器（直接從瀏覽器上傳到 Cloudinary）。

POST /api/upload：產生簽名，前端拿到後直接 POST 到 Cloudinary。

PATCH /api/upload：上傳完成後將 Cloudinary 回傳的 `public_id` + `secure_url` 寫回 Supabase：
- `type = 'main'`：更新 `main_photo` + `main_photo_public_id`
- `type = 'detail'`：將新物件 append 進 `detail_photos` 陣列

public_id 命名規則：
```
equipment-cards/{equipment_id}_{type}
範例：equipment-cards/1000003_main
      equipment-cards/1000003_detail_1
```
