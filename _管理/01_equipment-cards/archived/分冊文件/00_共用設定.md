# 00 共用設定

> 最後更新：2026-04-29

## 使用者背景

- 背景與經驗程度：有基礎，能自行使用 Claude Code 維護
- 主要使用場景：部門設備料卡管理，取代過大的 Excel 檔
- 預期讀者對象：自己（維護參考）＋ 團隊成員（操作說明）
- 作業系統／工具環境：Windows，Claude Code，Claude Cowork
- 語氣偏好：教學、淺顯易懂、白話為主

## 文件定位

- 主題：設備料卡管理系統開發專案
- 目的：開發參考手冊＋團隊操作說明
- 預期讀者：自用＋團隊

## 語氣範例

✅ 好的寫法（白話）：
> 「Supabase 就像是一個免費的雲端資料庫，我們把設備資料存在這裡，網頁就能隨時讀取。」

❌ 避免的寫法（過於技術）：
> 「Supabase 基於 PostgreSQL 提供 RESTful API 及 Realtime subscription 功能。」

## 術語對照表

| 術語 | 白話解釋 |
|------|----------|
| Schema | 資料庫的表格結構，定義有哪些欄位、存什麼資料 |
| Migration | 建立或修改資料庫表格的 SQL 指令檔 |
| Supabase | 免費的雲端資料庫服務，我們用它存設備資料 |
| Cloudinary | 免費的雲端照片儲存服務（25 credits/月，不需信用卡），取代原規劃的 Cloudflare R2 |
| Next.js | 我們用來做網頁的框架（工具） |
| Vercel | 免費的網站上線平台，把我們的網頁放到網路上 |
| RLS | Row Level Security，控制誰能看到哪些資料的安全機制 |
| Tailwind | 用來快速設計網頁外觀的 CSS 工具 |
| shadcn/ui | 現成的好看 UI 元件庫，直接拿來用 |
| PWA | 讓網頁可以像 App 一樣安裝在手機桌面 |
| R2 URL | 照片上傳到 R2 後取得的網址，存在資料庫裡 |
