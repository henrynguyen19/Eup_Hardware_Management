# 第 01 冊：專案總覽與技術選型

---

## 第 1 章 這個系統要解決什麼問題？

### 背景

部門原本用一個很大的 Excel 檔管理設備料卡。隨著設備越來越多、照片越來越大，Excel 開起來很慢、搜尋不方便、多人同時使用也容易衝突。

我們想用一個**網頁系統**來取代它，讓大家在瀏覽器打開就能用，不需要安裝任何軟體。

### 目標

- **使用人數**：部門 10～50 人
- **資料規模**：1,000～10,000 筆料卡，每筆有多張照片
- **核心功能**：
  - 照片牆瀏覽（像 Google 圖片那樣）
  - 關鍵字搜尋、模糊查詢
  - 依分類、廠商、狀態篩選
  - 排序功能
- **成本目標**：盡量免費，選用有免費方案的服務
- **維護方式**：由自己用 Claude Code 維護，不需要外包

## 第 2 章 我們用什麼工具？（技術選型）

### 2-1 整體架構圖

```
使用者的瀏覽器
     │
     ▼
  Next.js 網頁（部署在 Vercel）
     │
     ├─── 讀取資料 ──► Supabase（資料庫）
     │
     └─── 讀取照片 ──► Cloudflare R2（照片儲存）
```

簡單說：使用者打開網頁 → 網頁從 Supabase 拿設備資料 → 從 R2 拿照片 → 顯示出來。

### 2-2 各工具說明

#### Next.js 14（網頁框架）
- **是什麼**：做網頁的工具，幫我們處理畫面顯示和資料讀取
- **為什麼選它**：台灣工程師熟悉度高、社群資源多、跟其他工具整合容易
- **搭配 Tailwind + shadcn/ui**：快速做出好看的介面，不需要自己從頭設計 CSS

#### Supabase（資料庫）
- **是什麼**：免費的雲端資料庫，像 Google 試算表的進階版，但更強大、更安全
- **為什麼選它**：
  - 免費方案就夠用（500MB 資料庫、50,000 月活躍用戶）
  - 內建帳號登入功能（Auth），不需要自己寫
  - 台灣有很多教學資源
- **我們用它存什麼**：設備資料（名稱、分類、廠商、狀態、備註等）

#### Cloudflare R2（照片儲存）
- **是什麼**：免費的雲端照片/檔案儲存空間，類似 Google Drive 但給程式用的
- **為什麼選它**：
  - 免費方案每月 10GB 儲存、100 萬次讀取請求（截至 2025-05 為準）
  - 讀取費用為零（其他服務通常收流量費）
- **我們用它存什麼**：所有設備照片

#### Vercel（網站上線平台）
- **是什麼**：把我們做好的網頁放到網路上的平台
- **為什麼選它**：
  - 免費方案足夠部門內部使用
  - 跟 Next.js 是同一家公司，整合零障礙
  - 每次更新程式碼，自動重新部署

### 2-3 成本試算

| 服務 | 免費方案額度 | 我們的預估用量 | 是否足夠 |
|------|-------------|--------------|----------|
| Supabase | 500MB 資料庫 | 約 50MB（1萬筆文字資料） | ✅ 足夠 |
| Cloudflare R2 | 10GB 儲存 | 約 5～8GB（1萬張照片） | ✅ 足夠 |
| Vercel | 100GB 流量/月 | 部門內部使用量很低 | ✅ 足夠 |

> 💡 **初期幾乎零成本**。未來資料量超過免費額度時，才需要評估升級方案。

## 第 3 章 未來可以擴充什麼？

目前先做核心功能，以下功能**預留空間但不急著做**：

- **PWA（手機安裝）**：讓使用者把網頁加到手機桌面，像 App 一樣使用
- **廠商資料表**：目前廠商用文字欄位，未來可獨立成一張表，記錄聯絡人等資訊
- **狀態擴充**：目前只有「在線 / 停用」，未來可加「維修中」、「開發中」等
- **角色權限**：目前所有人看到一樣的內容，未來可分管理員和一般使用者
