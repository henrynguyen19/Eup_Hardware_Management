# 第 09 冊：認證與權限

---

## 1. 這章在說什麼

系統預設是「需要登入才能進入」。這章說明：

- 使用者怎麼用 **Google 帳號** 登入
- 系統怎麼知道你有沒有登入
- **管理員** 和 **一般使用者** 的權限差在哪
- 怎麼新增管理員、怎麼邀請新成員進來

> 💡 白話解釋：就像公司大門有門禁卡，這章說明門禁卡的設定方式和管理方法。

---

## 2. 系統架構總覽

```
使用者用瀏覽器開啟網址
         ↓
Next.js Server Component（伺服器端）
  → 查詢 Supabase：這個使用者有沒有登入？
  → 沒有 → 自動跳到 /login 登入頁
  → 有   → 查 profiles 表：是 admin 還是 viewer？
         ↓
     顯示首頁
  → admin：看到新增、編輯、刪除按鈕
  → viewer：只能瀏覽，看不到管理按鈕
```

### 用到的服務

| 角色 | 服務 | 說明 |
|------|------|------|
| 登入驗證 | Supabase Auth | 負責發 session（類似門票），記錄誰登入了 |
| OAuth 提供者 | Google | 讓使用者用 Google 帳號一鍵登入，不需要記密碼 |
| 路由保護 | Next.js Server Component | 每次進入首頁都確認 session 是否有效 |
| 角色管理 | Supabase `profiles` 表 | 記錄每個帳號是 admin 還是 viewer |

## 3. 首次設定流程（已完成，供日後參考）

這個設定只需要做**一次**，之後不需要重做。

### 3-1 設定 Supabase URL Configuration

> 告訴 Supabase：OAuth 登入完成後，要把使用者導回哪個網址。

1. 前往 [supabase.com](https://supabase.com) → 進入專案 `ntapfguwmuufnlafroxs`
2. 左側選單點 **Authentication**（鑰匙圖示）
3. 點 **URL Configuration**
4. 填入以下設定：

**Site URL**（主要網址）：
```
https://equipment-cards.vercel.app
```

**Redirect URLs**（允許的回調網址，兩條都要加）：
```
https://equipment-cards.vercel.app/auth/callback
http://localhost:3000/auth/callback
```

5. 按 **Save changes**

> 💡 `Redirect URLs` 只需要「存在清單裡」就會生效，不需要打勾。

---

### 3-2 在 Google Cloud Console 建立 OAuth 憑證

> 告訴 Google：「設備料卡管理系統」這個 App 是合法的，允許用 Google 帳號登入。

#### 步驟 1：前往 Google Cloud Console

開啟 [console.cloud.google.com](https://console.cloud.google.com)，確認右上角帳號是公司帳號（`@eup.com.tw`）。

#### 步驟 2：選擇或建立專案

上方會顯示目前的專案名稱（我們用的是 **equipment-cards**）。
如果找不到，點上方專案名稱 → 選「New Project」建立。

#### 步驟 3：進入 OAuth 設定

在頂部搜尋列輸入 **OAuth**，選擇「**OAuth 概覽**」（Google Auth Platform）。

#### 步驟 4：設定應用程式資訊（只做一次）

點「**開始**」→ 依序填寫：

| 欄位 | 填入內容 |
|------|----------|
| 應用程式名稱 | `設備料卡管理系統` |
| 使用者支援電子郵件 | 選你的公司 email |

按「**下一步**」。

#### 步驟 5：選擇目標對象

選擇「**內部**」。

> 💡 選「內部」的好處：只有 `@eup.com.tw` 組織內的 Google 帳號可以登入，不需要送審 Google 驗證，設定馬上生效。

按「**下一步**」直到完成設定。

#### 步驟 6：建立 OAuth 用戶端 ID

在 OAuth 總覽頁面，點「**建立 OAuth 用戶端**」。

| 欄位 | 填入內容 |
|------|----------|
| 應用程式類型 | 網頁應用程式 |
| 名稱 | `equipment-cards` |

往下：

**已授權的 JavaScript 來源**（點「+ 新增 URI」）：
```
https://equipment-cards.vercel.app
```

**已授權的重新導向 URI**（點「+ 新增 URI」）：
```
https://ntapfguwmuufnlafroxs.supabase.co/auth/v1/callback
```

按「**建立**」。

#### 步驟 7：取得 Client ID 和 Client Secret

建立後點進 `equipment-cards` 用戶端 → 點右上角的 **ⓘ 資訊圖示**。

| 項目 | 格式 |
|------|------|
| 用戶端 ID | `49182385706-xxxx...apps.googleusercontent.com` |
| 用戶端密鑰 | 點密鑰右邊的 📋 複製圖示取得（格式：`GOCSPX-xxxxxx`）|

---

### 3-3 將憑證填入 Supabase

1. 回到 Supabase → **Authentication → Sign In / Providers**
2. 往下找到 **Google** → 點進去
3. 填入：

| 欄位 | 填入內容 |
|------|----------|
| Client IDs | 貼上 Google 的「用戶端 ID」 |
| Client Secret (for OAuth) | 貼上 Google 的「用戶端密鑰」 |

4. 打開最上方的 **Enable Sign in with Google** 開關
5. 按 **Save**

> 完成後 Google 欄位會顯示 **Enabled** 綠燈。

## 4. 程式碼架構說明

> 這節給需要維護程式碼的人參考。

### 相關檔案

```
src/
├── lib/
│   ├── supabase-server.ts   ← Server Component / API Route 用的 Supabase client（讀 cookie）
│   ├── supabase-browser.ts  ← 瀏覽器端用的 Supabase client（UserMenu 登出用）
│   └── admin.ts             ← requireAdmin() / getUserRole() 兩個工具函式
├── app/
│   ├── page.tsx             ← 首頁：查詢 session → 未登入 redirect('/login')
│   ├── login/page.tsx       ← 登入頁：Google OAuth 按鈕
│   └── auth/callback/       ← OAuth 回調：把 Google 的 code 換成 session cookie
└── components/
    └── UserMenu.tsx         ← Header 右上角：顯示 email + 登出按鈕
```

### 路由保護原理

```typescript
// src/app/page.tsx（簡化說明）
const { data: { user } } = await supabase.auth.getUser()
if (!user) redirect('/login')   // 未登入 → 直接跳到登入頁
```

這段跑在 **伺服器端**（Node.js），比 middleware 更穩定可靠。使用者拿不到頁面內容，會直接被導走。

### 角色查詢

```typescript
// src/lib/admin.ts
async function getUserRole(): Promise<'admin' | 'viewer' | null>
async function requireAdmin()  // API route 用：不是 admin 就回 403
```

## 5. profiles 表結構

> `profiles` 表記錄每個登入帳號的角色。

```sql
CREATE TABLE profiles (
  id         UUID PRIMARY KEY REFERENCES auth.users(id),
  email      TEXT NOT NULL,
  role       TEXT DEFAULT 'viewer' CHECK (role IN ('admin', 'viewer')),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

| 欄位 | 說明 |
|------|------|
| `id` | 對應 Supabase Auth 的使用者 ID（自動關聯） |
| `email` | 登入時使用的 Google email |
| `role` | `admin`（管理員）或 `viewer`（一般使用者） |

**自動建立機制**：使用者**第一次**用 Google 帳號登入成功後，系統會自動在 `profiles` 表建立一筆資料，`role` 預設為 `viewer`。

## 6. 新增管理員

> 適用情境：某個使用者已經登入過（`profiles` 表有他的資料），要把他升為管理員。

**步驟：**

1. 前往 [supabase.com](https://supabase.com) → 專案 `ntapfguwmuufnlafroxs`
2. 左側點 **SQL Editor**
3. 輸入以下 SQL（把 email 換成該使用者的 Google email）：

```sql
UPDATE profiles
SET role = 'admin'
WHERE email = 'someone@eup.com.tw';
```

4. 按 **Run**
5. 看到 `Success` 即完成

> ⚠️ 注意：該使用者必須**已經登入過至少一次**，`profiles` 表才會有他的資料。若顯示「0 rows affected」，代表他尚未登入，請先請他登入一次。

**降回一般使用者：**

```sql
UPDATE profiles
SET role = 'viewer'
WHERE email = 'someone@eup.com.tw';
```

**查看目前所有帳號與角色：**

```sql
SELECT email, role, created_at FROM profiles ORDER BY created_at;
```

## 7. 邀請新成員 SOP

> 邀請同事加入系統的完整流程（約 5 分鐘）

### 前置條件

- 新成員有 `@eup.com.tw` 的 Google 帳號（公司 Google Workspace 帳號）
- 你有管理員權限，可以修改 Supabase

---

### 邀請步驟

#### 第一步：通知新成員

傳訊息給對方，提供以下資訊：

```
設備料卡系統網址：https://equipment-cards.vercel.app
登入方式：點「使用 Google 帳號登入」，選公司 Google 帳號（@eup.com.tw）
```

#### 第二步：請新成員登入一次

新成員自己：
1. 用 Chrome 開啟 `https://equipment-cards.vercel.app`
2. 點「**使用 Google 帳號登入**」
3. 選擇公司 Google 帳號（`@eup.com.tw`）
4. 登入後看到設備照片牆，即代表成功

> 這步很重要：必須**先登入一次**，系統才會在 `profiles` 表建立該帳號，你才能設定他的角色。

#### 第三步：確認帳號已建立（選做）

若要確認新成員是否已登入，到 Supabase → SQL Editor 執行：

```sql
SELECT email, role, created_at FROM profiles ORDER BY created_at DESC LIMIT 5;
```

看到新成員的 email 出現在清單裡即代表成功。

#### 第四步：若需要設為管理員

預設新成員是 `viewer`（只能瀏覽）。
若該成員需要管理員權限，執行：

```sql
UPDATE profiles
SET role = 'admin'
WHERE email = 'newmember@eup.com.tw';
```

## 8. 常見問題

**Q：使用者說「登入後看到錯誤畫面」怎麼辦？**

A：最常見原因是他用的 Google 帳號不是 `@eup.com.tw`（例如用個人 Gmail 登入）。請他登出後，重新選擇公司帳號登入。

**Q：想要讓系統允許 Gmail 帳號（非公司帳號）登入？**

A：目前系統使用 Google Workspace「內部」模式，只允許 `@eup.com.tw` 帳號。若要允許外部帳號，需要在 Google Cloud Console → OAuth 同意畫面改為「外部」，並向 Google 提交審核（較複雜，不建議）。

**Q：如何確認目前誰是管理員？**

A：Supabase → SQL Editor，執行：
```sql
SELECT email, role FROM profiles WHERE role = 'admin';
```

**Q：不小心把自己的帳號降成 viewer 怎麼辦？**

A：用 Supabase SQL Editor（用 service_role 登入，不受 RLS 限制）直接執行 UPDATE 改回來即可。

**Q：有人離職，要移除他的登入權限怎麼辦？**

A：目前最簡單的方式是去 Supabase → **Authentication → Users**，找到該帳號，點「Delete User」刪除。他下次嘗試登入就會失敗。
