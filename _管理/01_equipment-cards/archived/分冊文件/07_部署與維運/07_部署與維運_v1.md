# 第 07 冊：部署與維運

## 1. 部署架構

```
本機開發
    ↓ git push
GitHub（homejay-eup/equipment-cards）
    ↓ 自動觸發
Vercel（自動 build + deploy）
    ↓
https://equipment-cards.vercel.app
```

push 到 GitHub main branch 後，Vercel 自動偵測並在約 1–2 分鐘內完成部署，無需手動操作。

## 3. 環境變數

### 本機開發（.env.local）

```
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=dnqtafoh6
CLOUDINARY_API_KEY=
CLOUDINARY_API_SECRET=
CLOUDINARY_UPLOAD_FOLDER=equipment-cards
NEXT_PUBLIC_APP_URL=http://localhost:3000

# 選填：限制只有此網域的 Google 帳號可登入（例如 eup.com.tw）
# 不設定則允許所有 Google 帳號
ALLOWED_EMAIL_DOMAIN=eup.com.tw
```

`.env.local` 不 commit 到 GitHub（已加入 `.gitignore`）。`.env.local.example` 是範本，有欄位名稱但無值。

> ⚠️ **注意**：`ALLOWED_EMAIL_DOMAIN` 目前**不設定**也沒關係，因為系統已使用 Google Workspace「內部」模式，本身就限定 `@eup.com.tw`。此變數是額外的程式碼層防護，保留備用。

### Vercel 環境變數

Vercel Dashboard → 專案 → Settings → Environment Variables，需設定與 `.env.local` 相同的所有變數。新增或修改環境變數後需重新部署才生效。

## 4. 日常部署流程

```bash
# 修改程式碼後
git add .
git commit -m "說明修改內容"
git push
# → Vercel 自動部署，約 1–2 分鐘後上線
```

git commit 作者設定：
```bash
git config user.name "homejay-eup"
git config user.email "homejay@eup.com.tw"
```
換電腦時需重新執行上述設定。

## 5. 手動部署（備用）

自動部署失敗時使用：
```bash
npx vercel --prod
```

## 6. 本機開發

```bash
npm install        # 第一次或 package.json 有變更時執行
npm run dev        # 啟動開發伺服器 → http://localhost:3000
```

## 7. 系統更新方式

所有功能更新、bug 修復皆透過 **Claude Code** 進行：

1. 開啟 Claude Code，告知要修改的功能
2. Claude Code 修改程式碼
3. 確認結果無誤後 `git push`
4. Vercel 自動部署上線

常見更新情境：
- 新增分類選項：修改 `PhotoWall.tsx` 的 `CATEGORIES` 陣列
- 調整 UI 樣式：修改對應元件的 Tailwind class
- 新增功能（管理後台等）：依分冊手冊規劃執行
