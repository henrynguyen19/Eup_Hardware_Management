## 管理後台規劃（Step 7–9）

### Step 7：Supabase Auth + Google OAuth + 路由保護
- Supabase 開啟 Google provider
- Google Cloud Console 設定 OAuth credentials（一次性）
- Next.js middleware 保護 `/admin/*` 路由
- 登入頁 `/admin/login`：點「用 Google 帳號登入」→ 選公司帳號 → 進後台
- 限制只有 `@eup.com.tw` 網域才能登入

### Step 8：料卡管理 UI
- 料卡列表 + 搜尋
- 新增料卡（含主照片上傳）
- 編輯料卡（文字欄位 + 主照片替換）
- 細節照片管理（新增 / 刪除多張）
- 刪除料卡（同步刪 Cloudinary 照片）

### Step 9：使用者管理 UI
- 列出目前有後台權限的使用者
- 新增使用者（輸入 @eup.com.tw 信箱）
- 移除使用者
- 所有 admin 權限相同（無角色分級）

### Step 10：批次重整工具（Step 7–9 完成後執行）
- 清空腳本：清除 Supabase 所有料卡資料 + Cloudinary 所有照片
- 新版批次上傳腳本：讀取 CSV 對照表 + 照片資料夾 → 重新寫入
- CSV 欄位：品號、品名、分類、廠商、狀態、標籤、備註、主照片檔名、細節照片檔名
- 用途：大改重整時使用，日常維護用管理後台 CRUD
