> ✅ **本冊已完成**（Step 2 執行完畢）

## 第 1 章 整體架構說明

這一步的目標是把「資料要放哪裡」這件事設定好：

- **Supabase**：存設備的文字資料（名稱、廠商、備註…）
- **Cloudinary**：存照片檔案（主照片 + 細節照片）
- **.env.local**：把各服務的帳號金鑰統一放在這個檔案，程式從這裡讀取

## 第 2 章 Supabase 建表

### 建好的資料表結構（equipment_cards）

| 欄位 | 類型 | 說明 |
|------|------|------|
| equipment_id | TEXT | 設備 ID，如 `1000003`，對應照片檔名前綴 |
| name | TEXT | 設備名稱 |
| category | TEXT | 分類：主機、天線、支架、螢幕… |
| vendor | TEXT | 廠商名稱 |
| status | TEXT | `active`（現役）或 `discontinued`（停用） |
| tags | TEXT[] | 彈性標籤，如 `{4G, 含RFID}` |
| notes | TEXT | 用途說明、備註 |
| main_photo | TEXT | Cloudinary 主圖 URL |
| main_photo_public_id | TEXT | Cloudinary 主圖 public_id，刪除照片時使用 |
| detail_photos | JSONB | 細節照片陣列，格式見下方 |
| created_at | TIMESTAMPTZ | 建立時間 |
| updated_at | TIMESTAMPTZ | 最後更新時間（自動） |

**detail_photos 儲存格式：**
```json
[
  { "public_id": "equipment-cards/1000003_2",   "url": "https://res.cloudinary.com/…" },
  { "public_id": "equipment-cards/1000003_整組", "url": "https://res.cloudinary.com/…" }
]
```

### 執行步驟

1. 登入 [supabase.com](https://supabase.com) → 進入專案 → **SQL Editor**
2. 執行 `supabase/migrations/0001_create_equipment_cards.sql`
3. 執行 `supabase/migrations/0002_update_photo_fields_cloudinary.sql`

已建立的索引：
- `pg_trgm` 模糊搜尋索引（name、vendor、notes）
- category、status 精確篩選索引
- tags 陣列 GIN 索引

RLS 政策：**已登入使用者可讀寫**，匿名使用者無法存取。

## 第 3 章 Cloudinary 設定

> ⚠️ 原規劃使用 Cloudflare R2，因 Cloudinary 免費方案（25 credits/月，不需信用卡）更易上手，改採 Cloudinary。

### 帳號建立
前往 [cloudinary.com](https://cloudinary.com) 免費註冊，取得：

| 變數名稱 | 說明 | 取得位置 |
|----------|------|----------|
| `NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME` | Cloud name | Dashboard 右上角 |
| `CLOUDINARY_API_KEY` | API Key | Settings → Access Keys |
| `CLOUDINARY_API_SECRET` | API Secret | Settings → Access Keys |

### 上傳資料夾
所有照片存放在 `equipment-cards/` 資料夾下：
- 主照片：`equipment-cards/{equipment_id}_main`
- 細節照片：`equipment-cards/{equipment_id}_{suffix}`（suffix 如 `2`、`整組`、`配線`）

## 第 5 章 API Routes 設計

Next.js App Router 下建立的後端路由：

| 路由 | 方法 | 功能 |
|------|------|------|
| `/api/upload` | POST | 產生 Cloudinary 簽名，前端直接上傳（不過 Vercel） |
| `/api/upload` | PATCH | 上傳完成後將 public_id + URL 寫入 Supabase |
| `/api/upload/[id]` | DELETE | 刪除 Cloudinary 圖片並更新 Supabase |

Hook：`src/hooks/usePhotoUpload.ts` 封裝三步驟流程供前端元件使用。
