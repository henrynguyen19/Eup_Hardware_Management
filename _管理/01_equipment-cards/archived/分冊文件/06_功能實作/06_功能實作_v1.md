# 第 06 冊：功能實作

## 1. 搜尋與篩選架構

所有搜尋、篩選、排序邏輯都在 `PhotoWall.tsx` 的客戶端處理，不打後端 API。首頁一次載入全部 786 筆資料，之後的過濾全在瀏覽器記憶體中用 Fuse.js 完成。

## 2. Fuse.js 模糊搜尋

```typescript
const fuse = useMemo(() => new Fuse(initialCards, {
  keys: [
    { name: 'equipment_id', weight: 2 },
    { name: 'name',         weight: 2 },
    { name: 'vendor',       weight: 1 },
    { name: 'tags',         weight: 1 },
    { name: 'notes',        weight: 0.5 },
  ],
  threshold: 0.4,        // 0 = 完全比對，1 = 全部通過；0.4 可容許少量錯字
  includeScore: true,
  minMatchCharLength: 1,
}), [initialCards])
```

搜尋欄位與權重：
- `equipment_id`、`name`：權重 2（最優先）
- `vendor`、`tags`：權重 1
- `notes`：權重 0.5

使用者可直接輸入品號、品名、廠商、標籤名稱，支援模糊比對（例如輸入「格瑪」可找到「格瑪車機」）。

## 3. 篩選流程（四步驟）

```typescript
// 1. 模糊搜尋（有關鍵字）或全部
let result = query.trim()
  ? fuse.search(query.trim()).map(r => r.item)
  : [...initialCards]

// 2. 分類篩選
if (category !== '全部') {
  result = result.filter(c => c.category === category)
}

// 3. 狀態篩選
if (status !== 'all') {
  result = result.filter(c => c.status === status)
}

// 4. 排序（無搜尋關鍵字時才排序；有關鍵字時 Fuse 已按相關度排）
if (!query.trim()) {
  result.sort((a, b) =>
    sortBy === 'name'
      ? a.name.localeCompare(b.name, 'zh-TW')
      : a.equipment_id.localeCompare(b.equipment_id)
  )
}
```

## 7. URL 同步

篩選條件即時同步到 URL query string，方便分享特定篩選結果：

```typescript
useEffect(() => {
  const params = new URLSearchParams()
  if (query)              params.set('q',      query)
  if (category !== '全部') params.set('cat',    category)
  if (status   !== 'all') params.set('status', status)
  if (sortBy   !== 'id')  params.set('sort',   sortBy)
  const qs = params.toString()
  router.replace(qs ? `?${qs}` : '/', { scroll: false })
}, [query, category, status, sortBy, router])
```

URL 範例：
```
?q=格瑪&cat=主機&status=active
?cat=鏡頭&sort=name
```

頁面初始化時從 URL 讀取初始值，支援直接用連結開啟特定篩選狀態：

```typescript
const [query,    setQuery]    = useState(() => searchParams.get('q')      ?? '')
const [category, setCategory] = useState(() => searchParams.get('cat')    ?? '全部')
const [status,   setStatus]   = useState(() => searchParams.get('status') ?? 'all')
const [sortBy,   setSortBy]   = useState(() => searchParams.get('sort')   ?? 'id')
```

## 8. 清除篩選

任何篩選條件啟用時，顯示「× 清除篩選」按鈕：

```typescript
const hasActiveFilters = !!(query || category !== '全部' || status !== 'all')

const clearFilters = () => {
  setQuery('')
  setCategory('全部')
  setStatus('all')
  setSortBy('id')
}
```

## 9. 管理員後台（2026-05-03 新增）

### 9-1 功能說明

登入後系統會查詢 `profiles` 表確認角色：

- **admin（管理員）**：看到「＋ 新增料卡」按鈕（右下角）＋ 卡片 hover 出現「編輯」「刪除」按鈕
- **viewer（一般使用者）**：只能瀏覽，看不到任何管理按鈕

### 9-2 API Routes

| 路由 | 方法 | 說明 |
|------|------|------|
| `/api/cards` | POST | 新增料卡 |
| `/api/cards/[id]` | PATCH | 更新文字欄位 |
| `/api/cards/[id]` | DELETE | 刪除料卡（含 Cloudinary 所有照片）|

所有 API 都會先呼叫 `requireAdmin()`，非管理員回傳 `403 Forbidden`。

### 9-3 相關元件

| 元件 | 說明 |
|------|------|
| `CardFormDialog.tsx` | 新增／編輯料卡的表單 Dialog，含照片上傳管理 |
| `EquipmentCardItem.tsx` | 管理員 hover 時左上角顯示編輯／刪除按鈕 |
| `PhotoWall.tsx` | 接收 `isAdmin` prop，控制管理功能的顯示 |

### 9-4 照片上傳流程（管理員）

編輯 Dialog 內上傳照片的流程與批次上傳相同：

1. 前端向 `POST /api/upload` 取得 Cloudinary 簽名
2. 瀏覽器直接上傳到 Cloudinary（不經過 Vercel）
3. 上傳成功後，`PATCH /api/upload` 將 URL 寫回 Supabase

### 9-5 刪除料卡流程

`DELETE /api/cards/[id]` 的執行順序：

1. 從 Supabase 讀出該料卡的 `main_photo_public_id` 和 `detail_photos`
2. 用 `Promise.allSettled` 並行刪除所有 Cloudinary 照片
3. 從 Supabase 刪除料卡資料

> 使用 `allSettled`（而非 `all`）的原因：即使某張照片在 Cloudinary 已不存在，也不會中斷整個刪除流程。
